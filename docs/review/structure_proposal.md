### 1.2 構造上の問題点

#### 問題1: データが理論に先行する（Section 2 → Section 3）

現行構造の最大の問題は、**177ヶ国のマクロデータ分析（Section 2）が理論枠組み（Section 3）の前に配置されている**ことである。読者は以下の困難に直面する:

- 「第一近似R = I/E」「広告曝露プロキシ」といった変数が何を意味するのか、理論的根拠なしに突然導入される
- Block Aの「知見1: 相関構造が発展段階で相転移する」は、閾値モデル（Section 3.2.4）を知らなければ解釈できない
- データの「崩壊→精緻化→再構築」という探索的プロセスが、読者にとっては迷路のように感じられる
- 英語版プレプリントでは理論が先（Section 3）に配置されており、**日英間で構造が矛盾している**

学術論文の慣例として、理論仮説を先に提示し、その予測をデータで検証するのが標準的な流れである。現行構造は著者の研究プロセス（発見の順序）を反映しているが、**読者の理解プロセス**には最適化されていない。

#### 問題2: 個人レベルデータの分断

個人レベルの証拠が2箇所に分散している:

- **Section 2.3 (Block C)**: NHANES (N=5,032) と ATUS (N=21,736) の直接分析
- **Section 6**: 行動経済学・環境心理学・神経科学の既存知見の再解釈

Block Cは「マクロ分析の限界を補完する個人レベル検証」として配置されているが、Section 6の「収束的証拠」と本質的に同じ機能を果たしている。読者はSection 2.3で個人レベルの話に入り、Section 3-5でマクロ/理論に戻り、Section 6で再び個人レベルに戻るという往復を強いられる。

#### 問題3: 研究日記的叙述

Section 2全体が「第一近似 → 崩壊 → 診断バイアス発見 → ハードアウトカム検証 → 閾値掃引 → 広告プロキシ → 形式的検証」という時系列的な探索プロセスとして記述されている。これは著者の知的誠実さの表れだが、結果として:

- Block Aの知見がBlock Bで覆される（第一近似のTWFE崩壊）
- 読者が「最終的に何が言えるのか」を把握するまでに15ページ以上を要する
- 「崩壊は失敗ではなく教訓」という弁明が繰り返される

#### 問題4: 操作化と提案デザインの断絶

Section 4（操作化）とSection 5（提案デザイン）は、理論（Section 3）と反証条件（Section 7）の間に挟まれているが:

- Section 4のMAIA-2やTAS-20はSection 3.3のループ閉合度理論と密接に関連する
- Section 5のESMデザインはSection 7の反証条件と対になるべきもの
- Section 4.4ではATUS分析に言及するが、その結果はSection 2.3.3に記載されている（前方参照）

#### 問題5:「筆者の前提」セクションの異例さ

「筆者の前提と本論文の位置づけ」セクションは:

- 独立研究者であること、AI協働で執筆したことを開示
- 学術論文としては異例の配置（通常は脚注、Acknowledgments、またはDeclaration of Interestsで扱う）
- 開示内容自体は価値があるが、論文冒頭に配置することで読者の信頼を損なうリスクがある

## 2. 提案構造 Option A: Theory-First Approach（推奨）

### 2.1 構成概要

従来の学術論文の標準構成（Introduction → Theory → Methods → Results → Discussion）に準拠しつつ、探索的研究としての誠実さを保持する。

```
1. Introduction（導入）
   1.1 パラドックスの提示
   1.2 既存論争の構造的問題
   1.3 本論文の主張と貢献

2. Theoretical Framework（理論枠組み）
   2.1 栄養代謝アナロジーとバランスモデル
   2.2 認知負荷の非対称蓄積メカニズム
   2.3 数理的定式化 L = α₁·I − α₂·C
   2.4 感覚運動ループ閉合度による質的分類
   2.5 自己参照ループの一般病理学
   2.6 生成AIとしての究極的テストケース
   2.7 理論からの予測の要約

3. Macro-Level Evidence（マクロレベル証拠: Study 1）
   3.1 データと方法（177ヶ国パネル）
   3.2 主要知見
       3.2.1 発展段階による相関構造の相転移
       3.2.2 診断バイアスの発見とハードアウトカム検証
       3.2.3 Dose-Response反転: 保護効果から有害効果へ
       3.2.4 広告曝露プロキシの構築と検証
       3.2.5 閾値構造の形式的検証（Hansen PTR）
       3.2.6 認知的密度の修飾効果
   3.3 因果方向の検証: ラグ付きパネル分析
   3.4 探索プロセスの透明性に関する注記
   3.5 マクロ分析の限界

4. Individual-Level Evidence（個人レベル証拠）
   4.1 NHANES分析: PHQ-9と身体運動（N=5,032）
   4.2 ATUS分析: 体験的処理の2成分構造（N=21,736）
   4.3 収束的証拠: 既存知見の再解釈
       4.3.1 体験消費 vs 物質消費
       4.3.2 自然暴露の神経メカニズム（sgPFC-DMN）
       4.3.3 認知的飢餓の処方的概念
       4.3.4 運動介入RCTエビデンス
       4.3.5 自己効力感の再解釈
   4.4 バランスモデル vs 既存理論: 差分的予測

5. Operationalization and Proposed Design（操作化と検証設計）
   5.1 体験的処理の測定候補（MAIA-2, TAS-20, ESM）
   5.2 提案ESMデザイン
   5.3 最小コスト検証実験（実験A: 自然×スマホ, 実験B: 屋内運動vs自然）

6. Falsification Conditions（反証条件）
   6.1 反証条件6項目
   6.2 現時点の証拠の率直な評価

7. Discussion（議論）
   7.1 既存文献との位置づけ
   7.2 政策的含意
   7.3 AI設計への含意: 認知運動化の原理
   7.4 観察問題
   7.5 倫理的考慮
```

### 2.2 Option Aの利点

1. **読者が理論を理解してからデータを読める**: バランスモデル・ループ閉合度・閾値モデルが先に提示されるため、マクロデータの解釈が容易になる
2. **個人レベル証拠の統合**: Block C（NHANES/ATUS）とSection 6（収束的証拠）が一つのセクションにまとまり、マクロ→ミクロの自然な流れが生まれる
3. **操作化と検証設計の結合**: 測定方法（Section 4）と研究デザイン（Section 5）、反証条件（Section 7）の関連が明確になる
4. **探索プロセスの誠実さを保持**: Section 3.4として「探索プロセスの透明性に関する注記」を設け、第一近似の崩壊と精緻化の経緯を簡潔に記述する（現行の15ページ分のプロセス記述を2-3ページに凝縮）
5. **英語版プレプリントとの整合**: 英語版は既にtheory-firstの構成を採用しており、日本語版もこれに揃えることで一貫性が生まれる

## 3. 提案構造 Option B: Data-Driven Approach（データ先行を維持する場合）

### 3.1 構成概要

著者がデータ先行の構成にこだわる場合の改善案。探索的プロセスを維持しつつ、読みやすさを改善する。

```
1. Introduction（導入）
   1.1 パラドックスの提示
   1.2 既存論争の構造的問題
   1.3 本論文の主張（バランス仮説の要約を先出し）
   1.4 論文の構成と読み方のガイド ← 新設

2. Macro-Level Exploration（マクロレベル探索: Study 1）
   2.1 データと方法
   2.2 第一近似とその崩壊（簡潔に）
   2.3 ハードアウトカム検証と相転移の発見
   2.4 広告曝露プロキシの構築
   2.5 Dose-Response反転の形式的検証
   2.6 因果方向の検証
   2.7 マクロ分析から導出される仮説 ← 移行セクション

3. Theoretical Framework（理論枠組み）
   3.1 栄養代謝アナロジー
   3.2 認知負荷メカニズム（数理的定式化含む）
   3.3 感覚運動ループ閉合度
   3.4 自己参照ループの一般病理学
   3.5 生成AI

4. Individual-Level Evidence（個人レベル証拠）← Block Cと旧Section 6を統合
   4.1 NHANES分析（N=5,032）
   4.2 ATUS分析（N=21,736）
   4.3 既存知見の収束的再解釈
   4.4 差分的予測のまとめ

5. Operationalization, Design, and Falsification（操作化・設計・反証）← 3セクションを統合
   5.1 体験的処理の測定候補
   5.2 提案ESMデザインと最小コスト検証実験
   5.3 反証条件

6. Discussion（議論）
   6.1 既存文献との位置づけ
   6.2 政策的含意とAI設計
   6.3 観察問題と倫理的考慮
   6.4 現時点の証拠の率直な評価
```

### 3.2 Option Bの改善点（現行比）

1. **Section 1.4「論文の構成と読み方のガイド」新設**: データが先に来る理由を明示し、読者にナビゲーションを提供する
2. **マクロ分析の探索プロセスを圧縮**: 第一近似の崩壊の詳細（現行で約5ページ）を2ページ程度に凝縮し、Section 2.7「マクロ分析から導出される仮説」で理論への橋渡しを行う
3. **個人レベル証拠の統合**: Block CとSection 6を一つのセクションに統合
4. **操作化・設計・反証の統合**: 相互に関連する3セクションを一つにまとめる

### 3.3 Option Bの限界

- データが先に来る構成は学術論文として依然として非標準的であり、査読者から指摘を受ける可能性が高い
- Section 1.3で理論を「要約」として先出しする必要があるが、そうするとSection 3での詳述が冗長になるリスクがある
- 「探索プロセスの透明性」を維持しつつ読みやすさを確保するバランスが難しい

**最大の変更: 第一近似の崩壊プロセスの圧縮**

現行のSection 2.1-2.2は、以下の探索プロセスを逐次的に記述している:
1. 第一近似R = I/Eの構築（知見1-4）
2. Country FEでの頑健性確認
3. TWFEでの符号反転（崩壊）
4. 崩壊の構造分析（GDP診断バイアス）
5. ハードアウトカムによる検証
6. 閾値掃引
7. 広告プロキシの構築
8. 1階差分による識別
9. 形式的閾値検定

Option Aでは、このプロセスを**結果ベース**で再構成する:
- 最終的に支持される知見（広告プロキシ、相転移、dose-response反転）を主要結果として提示
- 第一近似の崩壊は「新Section 3.4: 探索プロセスの透明性に関する注記」で簡潔に記述（2-3ページ）
- 「崩壊した結果」は主要知見として報告しない

**新Section 3.4の内容**:
> 透明性の観点から、本研究の探索プロセスを記述する。分析は当初、Internet/Education比（R = I/E）を認知入力/処理容量の第一近似としたが、TWFE（Two-Way Fixed Effects）で符号が反転し、共通時間トレンドへの依存が露呈した（Table X参照）。この崩壊はGDPによる診断バイアスの発見（Section 3.2.2）を経て、広告曝露プロキシへの精緻化（Section 3.2.4）を動機づけた。再現リポジトリには第一近似の全分析結果を含む。

## 5. セクション間接続テキストのガイド

### 5.1 Introduction → Theoretical Framework（新Section 1 → 2）

### 5.2 Theoretical Framework → Macro-Level Evidence（新Section 2 → 3）

**新Section 2.7末尾に追加する接続テキスト**:
> 以上の予測をマクロレベルで検証するため、Section 3では177ヶ国×1990-2023年のパネルデータを用いた系統的分析を報告する。重要な注記として、本分析は上記の予測を事前登録的に検証したものではなく、理論の構築とデータ分析が相互作用的に進行した探索的プロセスである（プロセスの詳細はSection 3.4参照）。したがって、以下の結果は理論の確認ではなく、理論と整合的なパターンの提示として解釈されるべきである。

### 5.3 Macro-Level Evidence → Individual-Level Evidence（新Section 3 → 4）

## 6. 推奨と理由

### 6.1 推奨: Option A（Theory-First Approach）

以下の理由からOption Aを推奨する。

**理由1: 学術的標準への準拠**

探索的研究であっても、論文として公開する時点では理論→データの構成が標準的である。査読プロセスにおいて「なぜデータが理論の前にあるのか」という指摘は確実に生じる。

**理由2: 読者の認知負荷の低減**

皮肉なことに、「認知の肥満」を論じる論文が読者に過大な認知負荷を強いている。理論を先に提示することで、データセクションの各知見が「何を意味するのか」を読者が即座に理解できる。

**理由3: 英語版プレプリントとの整合性**

英語版（19ページ）は既にtheory-firstの構成を採用している。日本語版もこれに揃えることで、両版間の一貫性が確保され、国際的なレビュー・引用が容易になる。

**理由4: 探索的誠実さの保持**

Option Aは探索プロセスを隠蔽するのではなく、新Section 3.4「探索プロセスの透明性に関する注記」として明示的に記述する。再現リポジトリに第一近似の全分析を含めることで、透明性は完全に保持される。

**理由5: ページ数の削減**

探索プロセスの圧縮により、日本語版の推定ページ数は41ページから32-35ページ程度に削減される。これは学術誌の投稿規定に適合しやすくなる。

### 6.2 実装の優先順位

1. **最優先**: 「筆者の前提」をAcknowledgementsに移動（最小の変更で最大の改善）
2. **高優先**: Section 3（理論）をSection 2の前に移動
3. **高優先**: Block C（NHANES/ATUS）とSection 6（収束的証拠）を統合
4. **中優先**: マクロ分析の探索プロセスを結果ベースに再構成
5. **中優先**: Section 4/5/7の統合・再配置
6. **低優先**: 接続テキストの書き換え

### 6.3 「筆者の前提」の扱いに関する補足提案

現行の「筆者の前提」セクションに含まれる情報は、以下のように分散配置することを提案する:

| 現行の内容 | 移動先 | 理由 |
|---|---|---|
| 独立研究者であること | 著者所属欄 "Independent Researcher" | 国際標準 |
| AI協働の開示 | Acknowledgments + Methods冒頭 | AI利用の開示はMethodsに含めるのが近年の慣行 |
